Документация REST API

Этот документ предоставляет обзор доступных маршрутов и их функциональности.

  • Пациенты - /registration_office

    GET: Получить список пациентов.

    POST: Добавить нового пациента.

  • Пациент по ID - /registration_office/{id}

    GET: Получить данные пациента по ID.

    PUT: Обновить данные пациента по ID.

    DELETE: Удалить пациента по ID.

  • Врачи - /doctors

    GET: Получить список врачей.

    POST: Добавить нового врача.

  • Врач по ID - /doctors/{id}

    GET: Получить данные врача по ID.

    PUT: Обновить данные врача по ID.

    DELETE: Удалить врача по ID.

  • Специализации - /specializations

    GET: Получить список специализаций.

    POST: Добавить новую специализацию.

  • Специализация по ID - /specializations/{id}

    GET: Получить данные специализации по ID.

    PUT: Обновить данные специализации по ID.

    DELETE: Удалить специализацию по ID.

  • Кабинеты - /rooms

    GET: Получить список кабинетов.

    POST: Добавить новый кабинет.

  • Кабинет по ID - /rooms/{id}

    GET: Получить данные кабинета по ID.

    PUT: Обновить данные кабинета по ID.

    DELETE: Удалить кабинет по ID.

  • Записи - /appointments

    GET: Получить список записей.

    POST: Добавить новую запись.

  • Запись по ID - /appointments/{id}

    GET: Получить данные записи по ID.

    PUT: Обновить данные записи по ID.

    DELETE: Удалить запись по ID.
